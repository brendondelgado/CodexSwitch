# Sparkle Auto-Updater Integration Guide

This document describes how to add Sparkle (https://sparkle-project.org) auto-update
capability to CodexSwitch. Sparkle is the standard macOS framework for distributing
application updates outside the Mac App Store.

## 1. Add Sparkle via Swift Package Manager

In `Package.swift`, add the Sparkle dependency:

```swift
let package = Package(
    name: "CodexSwitch",
    platforms: [.macOS(.v15)],
    dependencies: [
        .package(url: "https://github.com/sparkle-project/Sparkle", from: "2.6.0"),
    ],
    targets: [
        .executableTarget(
            name: "CodexSwitch",
            dependencies: ["Sparkle"],
            path: "Sources/CodexSwitch",
            swiftSettings: [.swiftLanguageMode(.v6)]
        ),
        // ...
    ]
)
```

## 2. Info.plist Keys

Add these keys to the Info.plist generated by `scripts/build-app.sh`:

```xml
<!-- Sparkle auto-updater configuration -->
<key>SUFeedURL</key>
<string>https://yourdomain.github.io/codexswitch/appcast.xml</string>

<!-- Allow automatic background checks (every 24h by default) -->
<key>SUEnableAutomaticChecks</key>
<true/>

<!-- Public EdDSA key for verifying update signatures -->
<key>SUPublicEDKey</key>
<string>YOUR_EDDSA_PUBLIC_KEY_HERE</string>
```

## 3. Generate Signing Keys

Sparkle 2.x uses EdDSA (Ed25519) signing. Generate a keypair:

```bash
# Download Sparkle release tools
curl -LO https://github.com/sparkle-project/Sparkle/releases/latest/download/Sparkle-2.6.4.tar.xz
tar xf Sparkle-2.6.4.tar.xz

# Generate EdDSA keypair (stored in Keychain)
./bin/generate_keys

# Output will show:
#   Public key: <base64 string>  <-- put this in SUPublicEDKey
#   Private key is stored in your Keychain under "Sparkle EdDSA Key"
```

The private key stays in your macOS Keychain. Never export or commit it.
The public key goes into Info.plist as `SUPublicEDKey`.

## 4. Integrate SPUStandardUpdaterController

In `AppDelegate.swift`, add the updater controller:

```swift
import Sparkle

@MainActor
final class AppDelegate: NSObject, NSApplicationDelegate {
    // Add alongside existing properties
    private var updaterController: SPUStandardUpdaterController!

    func applicationDidFinishLaunching(_ notification: Notification) {
        // ... existing setup ...

        // Initialize Sparkle (checks for updates automatically on a schedule)
        updaterController = SPUStandardUpdaterController(
            startingUpdater: true,
            updaterDelegate: nil,
            userDriverDelegate: nil
        )
    }
}
```

To add a manual "Check for Updates" button in SettingsView:

```swift
// In SettingsView, accept the updater
var checkForUpdates: (() -> Void)?

// In the Settings section:
Button("Check for CodexSwitch Updates") {
    checkForUpdates?()
}
```

Wire it from AppDelegate:

```swift
let settingsView = SettingsView(
    onRemoveAllAccounts: { [weak self] in self?.removeAllAccounts() },
    checkForUpdates: { [weak self] in
        self?.updaterController.checkForUpdates(nil)
    }
)
```

## 5. Build and Sign the Update Archive

After building the app with `scripts/build-app.sh`:

```bash
# Create a zip of the .app bundle for the update
cd build/
ditto -c -k --sequesterRsrc --keepParent CodexSwitch.app CodexSwitch-1.1.0.zip

# Sign the zip with your EdDSA key
# (generate_appcast does this automatically, or manually):
./bin/sign_update CodexSwitch-1.1.0.zip
# Output: sparkle:edSignature="..." length="..."
```

## 6. Generate appcast.xml

Sparkle provides `generate_appcast` to build the feed automatically:

```bash
# Point it at a directory containing your signed .zip files
./bin/generate_appcast /path/to/releases/

# This creates/updates appcast.xml with entries like:
```

Example appcast.xml structure:

```xml
<?xml version="1.0" encoding="utf-8"?>
<rss version="2.0" xmlns:sparkle="http://www.andymatuschak.org/xml-namespaces/sparkle">
  <channel>
    <title>CodexSwitch Updates</title>
    <language>en</language>
    <item>
      <title>Version 1.1.0</title>
      <pubDate>Mon, 24 Mar 2026 12:00:00 +0000</pubDate>
      <sparkle:version>1.1.0</sparkle:version>
      <sparkle:shortVersionString>1.1.0</sparkle:shortVersionString>
      <sparkle:minimumSystemVersion>15.0</sparkle:minimumSystemVersion>
      <description><![CDATA[
        <h2>What's New</h2>
        <ul>
          <li>Added first-run onboarding</li>
          <li>Bug fixes and performance improvements</li>
        </ul>
      ]]></description>
      <enclosure
        url="https://yourdomain.github.io/codexswitch/releases/CodexSwitch-1.1.0.zip"
        length="4523000"
        type="application/octet-stream"
        sparkle:edSignature="BASE64_SIGNATURE_HERE"
      />
    </item>
  </channel>
</rss>
```

## 7. Host the Update Feed (GitHub Pages)

The simplest approach is to use GitHub Pages from the same repository:

### Directory structure

```
docs/
  updates/
    appcast.xml
    releases/
      CodexSwitch-1.0.0.zip
      CodexSwitch-1.1.0.zip
```

### Setup

1. Go to the GitHub repo Settings > Pages
2. Set source to "Deploy from a branch" > `main` > `/docs`
3. The feed URL becomes: `https://<user>.github.io/<repo>/updates/appcast.xml`
4. Set this URL as `SUFeedURL` in Info.plist

### Release workflow

```bash
# 1. Build the release
./scripts/build-app.sh

# 2. Create signed zip
cd build/
ditto -c -k --sequesterRsrc --keepParent CodexSwitch.app CodexSwitch-1.1.0.zip

# 3. Move to releases dir
cp CodexSwitch-1.1.0.zip ../docs/updates/releases/

# 4. Regenerate appcast
./bin/generate_appcast ../docs/updates/releases/
# This updates docs/updates/appcast.xml

# 5. Commit and push
git add docs/updates/
git commit -m "release: CodexSwitch v1.1.0"
git push
```

## 8. Code Signing Considerations

For Gatekeeper and notarization (required for public distribution):

1. **Developer ID certificate**: Obtain from Apple Developer Program ($99/year)
2. **Code sign the app**:
   ```bash
   codesign --force --deep --options runtime \
     --sign "Developer ID Application: Your Name (TEAM_ID)" \
     build/CodexSwitch.app
   ```
3. **Notarize**:
   ```bash
   # Create zip for notarization
   ditto -c -k --sequesterRsrc --keepParent build/CodexSwitch.app CodexSwitch.zip

   # Submit
   xcrun notarytool submit CodexSwitch.zip \
     --apple-id "your@email.com" \
     --team-id "TEAM_ID" \
     --password "@keychain:AC_PASSWORD" \
     --wait

   # Staple the ticket
   xcrun stapler staple build/CodexSwitch.app
   ```
4. **Hardened runtime**: Add `--options runtime` to codesign (required for notarization)

Without notarization, users will see "app is damaged" or "unidentified developer"
warnings on macOS Sequoia+.

## 9. Testing the Update Flow

1. Build v1.0.0, install it
2. Build v1.1.0, sign it, update the appcast
3. Launch v1.0.0 — Sparkle should detect the update and prompt
4. Verify the update installs and relaunches correctly

For local testing without a web server:

```bash
# Serve the updates directory locally
cd docs/updates
python3 -m http.server 8080

# Temporarily set SUFeedURL to http://localhost:8080/appcast.xml
```
